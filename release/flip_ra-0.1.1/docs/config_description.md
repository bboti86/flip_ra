In RetroArch, the `retroarch.cfg` file contains a specific set of fields (prefixed with `cheevos_`) that control how the system interacts with **RetroAchievements**. 

Since you are using **spruceOS** on a Miyoo Flip, most of these will be found in your primary configuration file at `/RetroArch/.retroarch/retroarch.cfg`.

### **Core Authentication & Connectivity**
These fields handle your identity and connection to the RetroAchievements servers.

| Field | Description |
| :--- | :--- |
| `cheevos_enable` | Set to `true` to enable the RetroAchievements system entirely. |
| `cheevos_username` | Your RetroAchievements.org username. |
| `cheevos_password` | Your plain-text password. **Note:** RetroArch usually replaces this with a token after the first successful login for security. |
| `cheevos_token` | A unique session key generated by the server. If this is present, the password field is often ignored. |

---

### **Gameplay & Logic Modes**
These fields define the "rules" under which you play and how much credit you get.

| Field | Description |
| :--- | :--- |
| `cheevos_hardcore_mode_enable` | **Crucial:** When `true`, it disables savestates, cheats, and slow-motion. Playing in Hardcore mode earns you double points and a special "Hardcore" border on your profile. |
| `cheevos_leaderboards_enable` | Enables or disables in-game leaderboards (e.g., fastest level completion or high scores). |
| `cheevos_richpresence_enable` | When `true`, it sends detailed info to the website (e.g., "Level 2, 3 lives left") so friends can see exactly what you’re doing in-game. |
| `cheevos_test_unofficial` | Enables "Unofficial" achievements. These are usually sets currently being developed or tested by the community and do not award points. |
| `cheevos_start_active` | A debug feature that starts a session with all achievements already "active" or unlocked locally (rarely used for normal play). |

---

### **Visuals & Notifications**
These fields control the "widgets" and pop-ups that appear while you play.

| Field | Description |
| :--- | :--- |
| `cheevos_verbose_enable` | Enables detailed on-screen notifications (e.g., "Logged in successfully," "Game has 45 achievements"). |
| `cheevos_badges_enable` | Shows the achievement's unique icon (badge) in the RetroArch menu list. |
| `cheevos_auto_screenshot` | Automatically takes a screenshot the exact moment an achievement is earned. |
| `cheevos_unlock_sound_enable` | Plays a notification sound when an achievement pops. |
| `cheevos_challenge_indicators` | Displays a small on-screen icon when a "Challenge" achievement is active (e.g., "Don't take damage"). The icon disappears if you fail the condition. |

---

### **Visibility Controls (Granular)**
Recent versions of RetroArch allow you to hide specific types of messages to keep the screen clean.

* `cheevos_visibility_account`: Shows/hides login success/failure messages.
* `cheevos_visibility_summary`: Shows/hides the summary of achievements remaining when you first boot a game.
* `cheevos_visibility_unlock`: Shows/hides the notification when you actually earn an achievement.
* `cheevos_visibility_mastery`: Shows/hides the "Mastery" notification when you finish an entire set.
* `cheevos_visibility_progress_tracker`: Shows/hides the on-screen progress for "measured" achievements (e.g., "Killed 45/100 enemies").
* `cheevos_visibility_lboard_start`: Shows/hides the message when a leaderboard attempt begins.
* `cheevos_visibility_lboard_submit`: Shows/hides the message when your score is submitted.
* `cheevos_visibility_lboard_trackers`: Shows/hides the small on-screen score tracker during a leaderboard run.

---

### **Widget Appearance**
If you want to move the notifications (useful for the Miyoo's small screen):

* `cheevos_appearance_anchor`: Sets where the widget appears (e.g., `0` for top-left, `3` for bottom-right).
* `cheevos_appearance_padding_h`: Horizontal distance from the edge.
* `cheevos_appearance_padding_v`: Vertical distance from the edge.

> [!TIP]
> If you are editing this file on your **Arch Linux** machine for your Miyoo, ensure the device is fully shut down before removing the SD card. RetroArch often overwrites the `.cfg` file upon closing, which can revert your manual edits if the system is still "live."
